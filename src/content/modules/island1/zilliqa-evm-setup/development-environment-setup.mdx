## Windows Users: WSL2 Setup (Recommended)

### Why WSL2 for Blockchain Development?

**Important Note:** According to the [official Hardhat documentation](https://hardhat.org/docs/getting-started#installation), using WSL2 on Windows is highly recommended for the best development experience.

**Benefits of WSL2:**
- **Better compatibility** with Node.js and npm packages
- **Faster file system operations** for compilation and testing
- **Linux-native tooling** that most blockchain tools are designed for
- **Better performance** for complex development workflows
- **Consistent environment** matching production deployments

**macOS and Linux users can skip this section** as they already have native Unix environments.

### Installing WSL2

1. **Open PowerShell as Administrator** (Right-click Start button â†’ Windows PowerShell (Admin))
2. **Enable WSL feature:**
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   ```
3. **Enable Virtual Machine Platform:**
   ```powershell
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
4. **Install WSL2**
   ```powershell
   wsl.exe --install
   ```
5. Restart your PC at this stage
6. **Open PowerShell as Administrator again** (Right-click Start button â†’ Windows PowerShell (Admin))
7. **Install Ubuntu**
   ```powershell
   wsl.exe --install Ubuntu
   ```
8. **Create user account** when prompted (this will be your Linux user)
9. **Update the system:**
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

### Deploy to Local Network

**Create a deployment script** for better control and debugging:

**Create `scripts/deploy.ts`:**
```bash

```

**Create the deployment script** (`scripts/deploy.ts`):
```typescript
import { network } from "hardhat";

async function main() {
  // Get the current network name from Hardhat runtime
  const networkName = process.env.HARDHAT_NETWORK || "localhost";
  console.log(`Deploying Counter contract to ${networkName}...`);
  
  try {
    const { viem } = await network.connect({
      network: networkName,
    });

    const publicClient = await viem.getPublicClient();
    const [walletClient] = await viem.getWalletClients();
    
    console.log("Deploying from account:", walletClient.account.address);
    
    // Show network info
    const chainId = await publicClient.getChainId();
    console.log("Network:", networkName);
    console.log("Chain ID:", chainId);
    
    // Get the Counter contract artifacts
    const counter = await viem.deployContract("Counter");
    
    console.log("âœ… Counter deployed to:", counter.address);
    
    // Call incBy with 5 (like in the Ignition module)
    console.log("Calling incBy(5)...");
    const incByTx = await counter.write.incBy([5n]);
    
    console.log("Transaction hash:", incByTx);
    
    // Wait for transaction and get receipt
    const receipt = await publicClient.waitForTransactionReceipt({ 
      hash: incByTx 
    });
    
    console.log("âœ… incBy(5) transaction confirmed in block:", receipt.blockNumber);
    
    // Read the current value
    const currentValue = await counter.read.x();
    console.log("Current counter value:", currentValue.toString());
    
    console.log("\nğŸ‰ Deployment completed successfully!");
    console.log("Contract address:", counter.address);
    console.log("Network:", networkName);
    console.log("Chain ID:", chainId);
    
  } catch (error) {
    console.error("âŒ Deployment failed:", error);
    process.exit(1);
  }
}

main().catch(console.error);
```

## Troubleshooting Common Issues

### Node.js Issues

**"node is not recognized":**
- Restart your computer after Node.js installation
- Check PATH environment variable includes Node.js directory
- Reinstall Node.js with "Add to PATH" option checked

### Hardhat Issues

**"hardhat command not found":**
- Use `npx hardhat` instead of just `hardhat`
- Install Hardhat globally: `npm install -g hardhat` (not recommended)
- Ensure you're in the project directory
- I had issues with ignition so used manual deploy scripts

### Network Connection Issues

**"Network connection failed":**
- Check your internet connection
- Verify RPC URLs are correct
- Try switching networks in MetaMask
- Check if the network is experiencing downtime

### Compilation Errors

**"Solidity compiler error":**
- Check Solidity version compatibility
- Ensure `evmVersion: "shanghai"` is set for Zilliqa
- Update Hardhat and dependencies: `npm update`
