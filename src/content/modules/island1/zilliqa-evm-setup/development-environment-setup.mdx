## Windows Users: WSL2 Setup (REQUIRED)

### Why WSL2 for Blockchain Development?

**‚ö†Ô∏è CRITICAL FOR WINDOWS USERS:** WSL2 (Windows Subsystem for Linux) is **REQUIRED** for this course, not optional. According to the [official Hardhat documentation](https://hardhat.org/docs/getting-started#installation), WSL2 on Windows is essential for proper blockchain development. Without WSL2, you will encounter compatibility issues and errors in future modules.

**Benefits of WSL2:**
- **Better compatibility** with Node.js and npm packages
- **Faster file system operations** for compilation and testing
- **Linux-native tooling** that most blockchain tools are designed for
- **Better performance** for complex development workflows
- **Consistent environment** matching production deployments

**macOS and Linux users can skip this section** as they already have native Unix environments.

### Installing WSL2

1. **Open PowerShell as Administrator** (Right-click Start button ‚Üí Windows PowerShell (Admin))
2. **Enable WSL feature:**
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   ```
3. **Enable Virtual Machine Platform:**
   ```powershell
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
4. **Install WSL2**
   ```powershell
   wsl.exe --install
   ```

### Understanding WSL Command Prompts

**What you'll see after WSL is installed:**

When you successfully enter WSL, your command prompt will change to show a **`$`** symbol (bash prompt), which looks like this:
```bash
username@computername:~$
```

**Important notes about using WSL:**
- Every time you open a new terminal or command prompt window, you'll start in Windows PowerShell
- To enter WSL, simply type `wsl` and press Enter
- You'll know you're in WSL when you see the `$` bash prompt
- All commands in this course that show `bash` code blocks should be run in the WSL bash prompt (after typing `wsl`)

**Example of entering WSL:**
```powershell
# Windows PowerShell prompt looks like:
PS C:\Users\YourName>

# Type wsl and press Enter:
PS C:\Users\YourName> wsl

# Now you're in WSL bash (notice the $ prompt):
username@computername:~$
```

### üíæ Critical VS Code Tip: Always Save Your Files!

**This is the #1 mistake beginners make in VS Code - not saving files before compiling or running commands.**

**How to know if a file needs saving:**
- Look at the file tab at the top - if you see a **white dot** next to the filename, the file is **NOT saved**
- An **X** (close button) means the file is saved and up to date

**Ways to save files:**
- **Single file:** Press **Ctrl+S** (Windows/Linux) or **Cmd+S** (Mac)
- **All open files:** Press **Ctrl+K** then **S**, or use **File ‚Üí Save All** from the menu
- **Enable auto-save:** File ‚Üí Preferences ‚Üí Settings ‚Üí search for "Auto Save" ‚Üí select "afterDelay"

**Why this matters for smart contract development:**
- ‚ùå Edit a contract without saving ‚Üí compile command runs on old code ‚Üí you'll be confused why your changes aren't working
- ‚ùå Edit `.env` file without saving ‚Üí deployment fails with wrong credentials
- ‚ùå Edit config files without saving ‚Üí settings don't take effect
- ‚ùå Edit scripts without saving ‚Üí old version runs

**Pro tip:** Get in the habit of using **File ‚Üí Save All** (Ctrl+K then S) before running any terminal command. This simple habit will save you hours of debugging!

**Node.js provides:**

- **Build tools** for compiling and deploying smart contracts

### Installation Process

#### For WSL2 Users (Windows with WSL2)

**Install Node.js via Node Version Manager (NVM) - Recommended:**

1. **Open your WSL2 Ubuntu terminal**
2. **Install curl if not already installed:**
   ```bash
   sudo apt install curl -y
   ```
3. **Install NVM:**
   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
   ```
4. **Restart your terminal or reload bash profile:**
   ```bash
   source ~/.bashrc
   ```
5. **Install Node.js 22.10+:**
   ```bash
   nvm install 22
   nvm use 22
   nvm alias default 22
   ```

#### For macOS and Linux Users

**Install Node.js via Node Version Manager (NVM) - Recommended:**

Follow the same NVM installation steps as WSL2 users above.

#### Verify Installation

**For all users (WSL2, macOS, Linux):**

**Open your terminal** (WSL2 Ubuntu for Windows, regular terminal for Mac/Linux)

**Check Node.js version:**
   ```bash
   node --version
   ```
   Should show v22.10.0 or higher

**Check npm version:**
   ```bash
   npm --version
   ```
   Should show 10.0.0 or higher

**Test npm functionality:**
   ```bash
   npm --help
   ```
   Should display npm help information

## Troubleshooting Common Issues

### Node.js Issues

**"node is not recognized" or "command not found":**
- Restart your terminal after Node.js installation
- Run `source ~/.bashrc` to reload your bash profile
- Verify NVM is installed: `nvm --version`
- Try installing Node again: `nvm install 22`

**"npm: command not found":**
- npm comes with Node.js, so reinstall Node: `nvm install 22`
- Verify with `which npm` to see if it's in your PATH

### WSL-Specific Issues (Windows Users)

**"wsl: command not found":**
- Ensure WSL2 is fully installed (see installation section)
- Restart your computer after WSL2 installation
- Open PowerShell as Administrator and run: `wsl --status`

**"Windows Subsystem for Linux has no installed distributions":**
- Restart your computer
- Open PowerShell as Administrator
- Run: `wsl.exe --install Ubuntu`
- Wait for Ubuntu to fully install

**"Cannot find module" or path errors:**
- Make sure you're running commands inside WSL (you should see the `$` prompt)
- Type `wsl` in your terminal to enter WSL
- Stay in the WSL file system (~/), not Windows drives (`/mnt/c/...`)

**VS Code not connecting to WSL:**
- Install the "WSL" extension by Microsoft
- Click the remote connection button (bottom-left corner)
- Select "New WSL Window"
- Restart VS Code if needed

### VS Code Issues

**Extensions not working:**
- Make sure you installed extensions in the WSL remote window, not locally
- Look for "WSL: Ubuntu" in the bottom-left corner of VS Code
- Reinstall extensions if needed

**Terminal not opening:**
- Go to View ‚Üí Terminal (or press Ctrl+`)
- If it opens in PowerShell, type `wsl` to enter bash
- Alternatively, open a new WSL window (remote connection button)
