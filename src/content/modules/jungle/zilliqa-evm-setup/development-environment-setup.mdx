## Windows Users: WSL2 Setup (Recommended)

### Why WSL2 for Blockchain Development?

**Important Note:** According to the [official Hardhat documentation](https://hardhat.org/docs/getting-started#installation), using WSL2 on Windows is highly recommended for the best development experience.

**Benefits of WSL2:**
- **Better compatibility** with Node.js and npm packages
- **Faster file system operations** for compilation and testing
- **Linux-native tooling** that most blockchain tools are designed for
- **Better performance** for complex development workflows
- **Consistent environment** matching production deployments

**macOS and Linux users can skip this section** as they already have native Unix environments.

### Installing WSL2

#### Step 1: Enable WSL2

1. **Open PowerShell as Administrator** (Right-click Start button â†’ Windows PowerShell (Admin))
2. **Enable WSL feature:**
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   ```
3. **Enable Virtual Machine Platform:**
   ```powershell
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
4. **Restart your computer**

#### Step 2: Set WSL2 as Default

1. **Download WSL2 Linux kernel update:** Visit https://aka.ms/wsl2kernel
2. **Install the update package**
3. **Open PowerShell as Administrator again**
4. **Set WSL2 as default:**
   ```powershell
   wsl --set-default-version 2
   ```

#### Step 3: Install Ubuntu

1. **Open Microsoft Store**
2. **Search for "Ubuntu"**
3. **Install "Ubuntu" (latest LTS version)**
4. **Launch Ubuntu** from Start menu
5. **Create user account** when prompted (this will be your Linux user)
6. **Update the system:**
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

### Configure VS Code for WSL2

1. **Install "WSL" extension** in VS Code by Microsoft
2. **Connect to WSL:** Click the remote connection button (bottom-left corner) â†’ "New WSL Window"
3. **Verify connection:** Terminal should show Linux prompt, not Windows

**All subsequent steps in this guide should be performed in WSL2 Ubuntu terminal** if you're on Windows.

### Installation Process

#### For WSL2 Users (Windows with WSL2)

**Install Node.js via Node Version Manager (NVM) - Recommended:**

1. **Open your WSL2 Ubuntu terminal**
2. **Install curl if not already installed:**
   ```bash
   sudo apt install curl -y
   ```
3. **Install NVM:**
   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
   ```
4. **Restart your terminal or reload bash profile:**
   ```bash
   source ~/.bashrc
   ```
5. **Install Node.js 22.10+:**
   ```bash
   nvm install 22
   nvm use 22
   nvm alias default 22
   ```

#### For macOS and Linux Users

**Install Node.js via Node Version Manager (NVM) - Recommended:**

Follow the same NVM installation steps as WSL2 users above.

1. **Visit the official Node.js website:** https://nodejs.org/en/download
2. **Choose "Windows Installer"** from the download options
3. **Select the LTS version** (should be 22.10 or higher)
4. **Download the 64-bit .msi installer** (most common for Windows)

**Installation steps:**
1. **Run the downloaded installer** (`node-v{version}-x64.msi`)
2. **Click "Next" through the welcome screen**
3. **Accept the license agreement**
4. **Choose installation folder** (default `C:\Program Files\nodejs\` is recommended)
5. **Select features to install** - keep all defaults:
   - âœ… **Node.js runtime**
   - âœ… **npm package manager**
   - âœ… **Add to PATH**
   - âœ… **Online documentation shortcuts**
6. **Enable tools for native modules** (recommended for some blockchain packages)
7. **Click Install** and wait for completion
8. **Restart your computer** to ensure PATH changes take effect

#### Verify Installation

**For all users (WSL2, macOS, Linux, Windows):**

## Step 5: Installing Hardhat Ignition

### What is Hardhat Ignition?

**Hardhat Ignition provides:**
- **Declarative deployments** - describe what you want deployed
- **Dependency management** - automatically handle deployment order
- **Resumable deployments** - recover from failed deployments
- **Network compatibility** - works seamlessly across different networks
- **Verification integration** - automatically verify contracts

### Installation

Hardhat Ignition is included in the Hardhat toolbox, but let's ensure it's properly configured:

1. **Verify installation:**
   ```bash

   ```

2. **If not installed, add it:**
   ```bash

   ```

### Understanding Ignition Modules

**Example deployment module** (`ignition/modules/Counter.ts`):

```typescript
import { buildModule } from "@nomicfoundation/hardhat-ignition/modules";

export default buildModule("CounterModule", (m) => {
  // Deploy the Counter contract
  const counter = m.contract("Counter");

  // Call the incBy function with parameter 5
  m.call(counter, "incBy", [5n]);

  // Return the contract for use in other modules
  return { counter };
});
```

## Troubleshooting Common Issues

### Node.js Issues

**"node is not recognized":**
- Restart your computer after Node.js installation
- Check PATH environment variable includes Node.js directory
- Reinstall Node.js with "Add to PATH" option checked

### Hardhat Issues

**"hardhat command not found":**
- Use `npx hardhat` instead of just `hardhat`
- Install Hardhat globally: `npm install -g hardhat` (not recommended)
- Ensure you're in the project directory

### Network Connection Issues

**"Network connection failed":**
- Check your internet connection
- Verify RPC URLs are correct
- Try switching networks in MetaMask
- Check if the network is experiencing downtime

### Compilation Errors

**"Solidity compiler error":**
- Check Solidity version compatibility
- Ensure `evmVersion: "shanghai"` is set for Zilliqa
- Update Hardhat and dependencies: `npm update`
