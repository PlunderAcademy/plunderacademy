You've worked with **ERC20 tokens** (fungible) and **staking contracts** (time-based logic). NFTs introduce **uniqueness and individuality**:

**Mental model:**
- **ERC20**: Like a bank account - you care about *how many* tokens
- **Staking**: Like a savings account - you care about *when* you deposited
- **ERC721**: Like a trading card collection - you care about *which specific* tokens

## Key Concepts to Remember

### From ERC20 (Island 1, Module 5):
- **OpenZeppelin inheritance** (using battle-tested contracts)
- **Constructor parameters** (name, symbol, initial values)
- **Owner controls** (Ownable pattern)
- **Events** (emit for important state changes)

### From Staking Contract (Island 2, Module 5):
- **Hardhat 3 with Viem** (new testing patterns)
- **Zilliqa configuration** (shanghai EVM, specific chain IDs)
- **Environment variables** (.env for private keys and addresses)
- **Claimant tracking** (for achievement verification)
- **Deployment scripts** (automated, verifiable deployment)

### New for NFTs:
- **Token IDs** (each NFT has a unique ID: 1, 2, 3, ... 100)
- **Metadata URIs** (IPFS links to JSON describing each NFT)
- **Reveal mechanics** (hidden placeholder â†’ revealed metadata)
- **Paid minting** (users send ZIL to mint)
- **IPFS storage** (decentralized, permanent metadata hosting)

## NFTs vs Your Previous Work

### Similarities to ERC20:
- Both use OpenZeppelin base contracts
- Both have supply caps (totalSupply)
- Both emit Transfer events
- Both use Ownable for admin controls

### Similarities to Staking:
- Both accept payments (staking: tokens, NFT: ETH)
- Both track per-user data (staking: deposits, NFT: owned tokens)
- Both need claimant integration
- Both use same Hardhat 3 + Zilliqa setup

### Unique to NFTs:
- **Individual token tracking** (not balances)
- **Metadata per token** (images, traits, descriptions)
- **External storage** (IPFS for metadata/images)
- **Reveal mechanics** (time-delayed metadata switch)
- **Marketplace integration** (OpenSea, Rarible, etc.)

## Development Philosophy

We follow the same principles as the staking contract:

**1. Security First**
- Use OpenZeppelin's audited contracts
- Validate all inputs
- Prevent reentrancy
- Cap supply to prevent inflation

**2. User Experience**
- Clear error messages
- Refund excess payments
- Safe minting (prevent lost NFTs)
- Deterministic metadata

**3. Transparency**
- Emit events for all state changes
- Open source code
- Verifiable on block explorer
- IPFS for permanent metadata

**4. Production Ready**
- Tested thoroughly
- Gas optimized
- Marketplace compatible
- Standards compliant

## Common Questions

**Q: Do I need to create 100 unique images manually?**
A: No! We provide a script that auto-generates 100 colorful SVG shapes. Takes ~5 seconds.

**Q: What if I want to use PNG/JPG instead of SVG?**
A: You can! Lesson 3.7.4 covers how to upload any image type to IPFS.

**Q: Can I customize the MAX_SUPPLY?**
A: Yes! Change the constant in the contract and adjust the metadata generation scripts.

**Q: Do I need to pay for IPFS hosting?**
A: No! We'll use Pinata's free tier (1 GB free, plenty for this collection).

**Q: How much does it cost to deploy on Zilliqa testnet?**
A: Testnet ZIL is free from the faucet. Deployment costs ~$0 (testnet gas).

**Q: Will this work on Ethereum or other chains?**
A: Yes! The contract is EVM-compatible. Just change the Hardhat network configuration.

**Q: Can I sell these NFTs?**
A: These are testnet NFTs for learning. For mainnet, you'd deploy to Zilliqa mainnet.

**Q: What's different from the staking contract?**
A: Staking accepts ERC20 deposits. NFTs mint unique tokens for ZIL payments. Staking tracks time. NFTs track ownership.
